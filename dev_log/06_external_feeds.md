# Unit 06: External RSS Feed Integration

## Objective

Integrate external RSS feeds for software and football sections at build time, displaying title + paragraph + feed content exactly like original allende.nz implementation. Hard fail deployment if feeds unavailable.

## Implementation

### Technical Approach

1. **RSS Feed Integration**
   - Separate RSS feeds for software and football sections
   - Build-time fetching during Astro static generation
   - Hard failure if external feeds are unavailable (stops GitHub Actions deploy)
   - Display format: title + paragraph + extracted feed content

2. **Feed Processing**
   - Fetch RSS feeds during build process
   - Parse and extract relevant content (title, description, link, date)
   - Integrate seamlessly with existing autogenerated content
   - Maintain existing section structure and styling

3. **Error Handling**
   - Build fails if RSS feeds cannot be fetched
   - No fallback content - deployment stops on feed failure
   - GitHub Actions will not deploy if build fails
   - Clear error messages for debugging feed issues

4. **Content Display**
   - Preserve existing section content (title + paragraph)
   - Append RSS feed content below autogenerated content
   - Consistent styling with portfolio theme
   - Responsive design for feed items

### Key Decisions

- **Build-Time Only**: No client-side fetching, all RSS content static at build
- **Hard Failure Strategy**: Build fails if feeds unavailable, preventing deployment
- **No Combined RSS**: Disable existing RSS generation, keep feeds separate
- **Original Format**: Match allende.nz display format exactly
- **GitHub Actions Integration**: Leverage existing deployment pipeline failure handling
